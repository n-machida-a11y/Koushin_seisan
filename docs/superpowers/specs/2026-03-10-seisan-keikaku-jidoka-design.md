# 生産計画作成自動化 設計ドキュメント

作成日: 2026-03-10

---

## 1. 背景・目的

月1回、担当者1名が手作業で行っている生産計画作成業務（26ステップ）を自動化する。
現在はExcelの関数・手作業で実施しており、完成ファイルを共有フォルダで全社員に展開している。
見た目の変更は最小限に抑えつつ、将来的には誰でも実行できる形にする。

---

## 2. 全体アーキテクチャ

### アプローチ: VBA × UiPath ハイブリッド

- **VBA**: Excelの加工処理ロジック（ステップ⑤〜㉖）を担当
- **UiPath**: VBAの呼び出し役として一本化。将来的にシステムUI操作（①〜④）を追加する

```
[UiPath メインフロー]
    │
    ├─ Phase 1（現在）: Excel加工の自動化
    │       ├─ ①〜③ BHプランをPドライブへ保存（手動）
    │       ├─ ④ 更新システムのUI操作（手動 → Phase 2でUiPath化）
    │       └─ ⑤〜㉖ VBAマクロを呼び出してExcel加工を実行
    │
    └─ Phase 2（将来）: システム操作を追加
            └─ ①〜④のUI操作をUiPathフローの先頭に組み込む
```

### ファイル構成

```
Pドライブ/生産計画自動化/
  ├─ 生産計画_マクロ.xlsm     ← VBAマクロ本体 + 設定シート群
  └─ input/                   ← BHプランなどの入力ファイル置き場
```

---

## 3. VBAマクロ構成

### メイン実行

各ステップを独立したSubルーチンとして実装し、`メイン実行()`から順番に呼び出す。
各Subは単独でもテスト実行できるようにして、部分的な修正・再実行が容易な設計にする。

```vba
Sub メイン実行()
    Call 設定読み込み()
    Call Step05_計画生産対象削除()
    Call Step06_出荷済みデータ削除()
    Call Step07_型式補完()
    Call Step08_計画生産行展開()
    Call Step09_数量チェック()
    Call Step10_並び替え()
    ' ... 以降続く
    Call Step26_KMP_MPS更新()
    Call ログ出力("完了")
End Sub
```

## 5. エラー・例外処理

### 処理停止ケース（再開不要・最初からやり直し）

| ケース | 動作 |
|---|---|
| 3ヶ月以内に型式（S列）が空欄で自動補完不可 | 該当行を黄色ハイライト → 処理停止 → ポップアップ表示 |
| 3ヶ月以内データで数量が1でない（⑨） | 同上 |
| ファイルが見つからない等の実行時エラー | 即時停止 → エラー内容をポップアップ表示 |

### 警告で続行するケース

| ケース | 動作 |
|---|---|
| 4ヶ月以降の型式空欄 | 自動補完して続行 → ログシートに記録 |

## 7. UiPath連携（Phase 1）

Phase 1ではUiPathはVBAの呼び出し役に徹する。

```
UiPath フロー（Phase 1）
  1. 生産計画_マクロ.xlsm を開く
  2. "メイン実行" マクロを呼び出す
       （Excel Application Scope + Invoke VBA アクティビティ）
  3. エラーポップアップが出たら処理待機
  4. 完了後、ログシートの内容をメール通知（将来対応）
```

Phase 2ではこのフローの先頭に①〜④のシステムUI操作を追加するだけで完結する。

## 8. 実装フェーズ

| フェーズ | 内容 | 優先度 |
|---|---|---|
| Phase 1-A | 設定シート群の作成 ＋ ステップ⑤〜⑩（削除・補完・展開・並替） | 最優先 |
| Phase 1-B | ステップ⑪〜㉖（星取表・集計・KMP計画） | Phase 1-A完了後 |
| Phase 1-C | UiPath フロー（VBA呼び出し一本化） | Phase 1-B完了後 |
| Phase 2 | UiPath + 更新システムUI操作（①〜④）を追加 | 将来 |
| Phase 3 | 平準化へのAI活用検討 | 将来 |

## 9. 制約・注意事項

- フィリピン工場がプロダクションスケジュールをそのままシステムに取り込んでいるため、**フォーマット変更は厳禁**
- VPN接続中はインターネット遮断されるが、ローカルのExcel・UiPath・VBAは問題なく動作する
- 全社員に展開しているファイルの見た目は変更しない
