# VBAモジュール インポート手順

`生産計画_マクロ.xlsm` にVBAコードをインポートする手順です。
初回セットアップ時に一度だけ実施してください。

---

## 前提条件

- Windows Excel（2016以降推奨）
- 「開発」タブが表示されていること
  - 表示されていない場合: ファイル → オプション → リボンのユーザー設定 → 「開発」にチェック

---

## インポート手順

### 1. `生産計画_マクロ.xlsm` を Excel で開く

### 2. VBAエディタを開く
キーボードショートカット: `Alt + F11`

### 3. 以下の順番でモジュールをインポートする

VBAエディタのメニュー: **ファイル → ファイルのインポート**

インポートするファイル（`vba/` フォルダ内）と順番:

| 順番 | ファイル名 | 内容 |
|------|-----------|------|
| 1 | ModConfig.bas | 設定読み込み（基盤） |
| 2 | ModLog.bas | ログ書き込み（基盤） |
| 3 | ModError.bas | エラー処理（基盤） |
| 4 | ModStep05.bas | ステップ⑤: 計画生産対象削除 |
| 5 | ModStep06.bas | ステップ⑥: 出荷済みデータ削除 |
| 6 | ModStep07.bas | ステップ⑦: 型式補完 |
| 7 | ModStep08.bas | ステップ⑧: 計画生産行展開 |
| 8 | ModStep09.bas | ステップ⑨: 数量チェック |
| 9 | ModStep10.bas | ステップ⑩: 並び替え |
| 10 | ModMain.bas | メイン実行 |

### 4. 実行ボタンを追加する

1. Excel の「設定」シートを開く
2. 「開発」タブ → 「挿入」→「ボタン（フォームコントロール）」を選択
3. シート上の適当な場所にボタンを描画する
4. マクロの割り当てダイアログで `メイン実行` を選択して「OK」
5. ボタン上で右クリック → 「テキストの編集」→「▶ 生産計画自動化 実行」と入力

### 5. マクロを有効にして保存する

- ファイル → 名前を付けて保存 → ファイルの種類: **Excel マクロ有効ブック (*.xlsm)**
- セキュリティ警告が出た場合: 「コンテンツの有効化」をクリック

---

## 毎月の実行手順

1. BHプランの出力ファイル（xlsx）を `input\` フォルダに配置する
2. `生産計画_マクロ.xlsm` を開く
3. 「設定」シートの **「▶ 生産計画自動化 実行」** ボタンをクリック
4. 確認ダイアログで「はい」をクリック
5. 処理完了後、「ログ」シートで結果を確認する

---

## 設定変更が必要な場合

「設定」シートの B 列の値を直接書き換えてください。コードの変更は不要です。

よく変更する項目:
- `BHプラン保存フォルダ` → BHプランを置くフォルダのパス
- `BH計画保存版_V8パス` / `BH計画保存版_V9パス` → 保存版ファイルのパス
- `BH計画保存版_V8_KPNo列番号` / `BH計画保存版_V9_KPNo列番号` → 実際のファイルのKP-No列番号
- `問い合わせ先メール` → オムロン担当者のメールアドレス

---

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| 「マクロが見つかりません」エラー | VBAモジュールのインポートが完了しているか確認 |
| 「inputフォルダにxlsxが見つかりません」 | 設定シートの `BHプラン保存フォルダ` のパスとファイルの配置を確認 |
| 「シートが見つかりません」 | 設定シートの `加工対象シート名` が実際のシート名と一致しているか確認 |
| 処理が途中で停止した | ログシートと黄色ハイライトされた行を確認し、オムロン担当者に問い合わせ後、データ修正して再実行 |
